# Investor Update Template

## Purpose

The fillable recurring update [INVESTOR-UPDATES.md](../08-investors/INVESTOR-UPDATES.md) requires — progress, metrics, challenges, and asks, on a predictable cadence.

## When to Use

On the stated update cadence ([INVESTOR-UPDATES.md](../08-investors/INVESTOR-UPDATES.md)) — sent even when the news isn't all positive.

## Owner

Leadership, per [INVESTOR-OPERATING-SYSTEM.md](../08-investors/INVESTOR-OPERATING-SYSTEM.md).

## Inputs

Current metrics ([INVESTOR-METRICS.md](../08-investors/INVESTOR-METRICS.md), [BUSINESS-METRICS.md](../01-business/BUSINESS-METRICS.md)); progress against milestones in [FUNDRAISING-STRATEGY.md](../08-investors/FUNDRAISING-STRATEGY.md) or [GOALS-AND-OKRS.md](../01-business/GOALS-AND-OKRS.md).

## Template Section

```markdown
# Investor Update — <Company> — <Date>

## Headline
<One sentence: the single most important thing this period.>

## Key Metrics
| Metric | This Period | Last Period |
|--------|--------------|----------------|
| <metric> | <value> | <value> |

## Progress Against Milestones
<Specific progress toward the milestones this raise/period is meant to fund.>

## Wins
- <...>

## Challenges
<Stated honestly — what's hard right now, and what's being done about it.>

## Asks
<Specific: an introduction, advice, a hire — not vague "any help appreciated.">

## Next Update
<Date>
```

## Review Checklist

- [ ] Metrics match the same numbers tracked internally — no separately-flattering version.
- [ ] Challenges are stated honestly, not omitted.
- [ ] Asks are specific, not generic.
- [ ] Any financial/legal/compliance claim is verified per [TRUST-COMMUNICATIONS-AND-CLAIMS.md](../14-security-compliance/TRUST-COMMUNICATIONS-AND-CLAIMS.md).
- [ ] No placeholder text remains.
