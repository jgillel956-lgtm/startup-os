# Startup OS v1 Usage Guide

The entry point for a user starting to use Startup OS after this v1 buildout — a short pointer to the right existing guide for the right situation, not a restatement of any of them. Every guide below already exists and is maintained on its own; this document's only job is routing a new user to the correct one quickly.

## Start Here, Based on What You're Doing

| Situation | Go to |
|-----------|-------|
| Starting a brand-new company, product, or portfolio project from scratch, right now | [EXAMPLE-COMPANY-IMPLEMENTATION-GUIDE.md](EXAMPLE-COMPANY-IMPLEMENTATION-GUIDE.md) — the concrete, step-by-step first pass |
| Deciding when to bring in each domain as an existing company matures | [STARTUP-OS-ADOPTION-GUIDE.md](STARTUP-OS-ADOPTION-GUIDE.md) — phased adoption over time |
| Your company doesn't fit the default step order (sales-led, engineering-first, etc.) | [STARTUP-OS-IMPLEMENTATION-PATHS.md](STARTUP-OS-IMPLEMENTATION-PATHS.md) — how the order shifts by archetype |
| Checking what the minimum viable set of documents actually is | [COMPANY-OPERATING-PACKAGE.md](COMPANY-OPERATING-PACKAGE.md) — the minimum artifact bill-of-materials |
| Working through setup as a literal check-off list | [NEW-COMPANY-SETUP-CHECKLIST.md](NEW-COMPANY-SETUP-CHECKLIST.md) |
| Finding a specific document for a specific task ("I need to write a PRD") | [DOCUMENTATION-NAVIGATION-GUIDE.md](DOCUMENTATION-NAVIGATION-GUIDE.md) — the "I'm trying to..." lookup table |

## The One Rule That Applies Regardless of Where You Start

Read [AI_CONTEXT.md](../../AI_CONTEXT.md) (if an AI assistant is doing the work) and this domain's own [README.md](README.md) before anything else — every guide above assumes the foundation standards are already in force, per [STARTUP-OS-ADOPTION-GUIDE.md](STARTUP-OS-ADOPTION-GUIDE.md)'s "Don't skip Foundation."

## If You're Not Sure Which Guide Fits

Default to [EXAMPLE-COMPANY-IMPLEMENTATION-GUIDE.md](EXAMPLE-COMPANY-IMPLEMENTATION-GUIDE.md) for a new company and [DOCUMENTATION-NAVIGATION-GUIDE.md](DOCUMENTATION-NAVIGATION-GUIDE.md) for a specific task inside one already underway — between those two, nearly every real situation is covered.
