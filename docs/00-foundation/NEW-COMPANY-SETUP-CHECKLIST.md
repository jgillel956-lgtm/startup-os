# New Company Setup Checklist

A check-off version of [EXAMPLE-COMPANY-IMPLEMENTATION-GUIDE.md](EXAMPLE-COMPANY-IMPLEMENTATION-GUIDE.md)'s ten-step flow. This checklist doesn't re-explain each step — it exists to be worked through, not read as a standard. If a box doesn't make sense, the guide it's drawn from has the explanation.

## Phase 1 — Idea and Strategy

- [ ] One or two sentences stating what the company does and for whom.
- [ ] [COMPANY-STRATEGY.md](../01-business/COMPANY-STRATEGY.md) filled in from [BUSINESS-STRATEGY-TEMPLATE.md](../10-templates/BUSINESS-STRATEGY-TEMPLATE.md).
- [ ] [BUSINESS-MODEL.md](../01-business/BUSINESS-MODEL.md) drafted.
- [ ] First-pass [GOALS-AND-OKRS.md](../01-business/GOALS-AND-OKRS.md) named, even provisionally.
- [ ] [MARKET-AND-CUSTOMER-STRATEGY.md](../01-business/MARKET-AND-CUSTOMER-STRATEGY.md) filled in.

## Phase 2 — Product and Design

- [ ] First capability documented in [CAPABILITY-MODEL.md](../02-product/CAPABILITY-MODEL.md).
- [ ] First PRD written per [PRD-STANDARD.md](../02-product/PRD-STANDARD.md), with acceptance criteria per [ACCEPTANCE-CRITERIA-STANDARD.md](../02-product/ACCEPTANCE-CRITERIA-STANDARD.md).
- [ ] [UX-RESEARCH.md](../05-design/UX-RESEARCH.md) findings recorded, however minimal.
- [ ] Design brief created from [DESIGN-BRIEF-TEMPLATE.md](../10-templates/DESIGN-BRIEF-TEMPLATE.md).
- [ ] Design handed off per [DESIGN-HANDOFF.md](../05-design/DESIGN-HANDOFF.md).

## Phase 3 — Engineering

- [ ] Technical design written per [TECHNICAL-DESIGN.md](../03-engineering/TECHNICAL-DESIGN.md).
- [ ] Any consequential decision recorded as an ADR per [ARCHITECTURE-DECISION-RECORDS.md](../03-engineering/ARCHITECTURE-DECISION-RECORDS.md).
- [ ] Repository set up per [REPOSITORY-STANDARDS.md](../03-engineering/REPOSITORY-STANDARDS.md).
- [ ] First release planned per [RELEASE-AND-DEPLOYMENT.md](../03-engineering/RELEASE-AND-DEPLOYMENT.md).

## Phase 4 — Go-to-Market Basics

- [ ] [POSITIONING-AND-MESSAGING.md](../06-marketing/POSITIONING-AND-MESSAGING.md) written against a real, shipped (or nearly shipped) capability.
- [ ] [SALES-PIPELINE.md](../07-sales/SALES-PIPELINE.md) and [IDEAL-CUSTOMER-AND-QUALIFICATION.md](../07-sales/IDEAL-CUSTOMER-AND-QUALIFICATION.md) stood up.
- [ ] [MARKETING-TO-SALES-HANDOFF.md](../06-marketing/MARKETING-TO-SALES-HANDOFF.md) defined before the first qualified lead needs to move.

## Phase 5 — Support and Customer Success

- [ ] [CUSTOMER-SUPPORT-MODEL.md](../09-operations/CUSTOMER-SUPPORT-MODEL.md) and [SLA-AND-RESPONSE-STANDARDS.md](../09-operations/SLA-AND-RESPONSE-STANDARDS.md) in place before the first support request.
- [ ] [CUSTOMER-ONBOARDING.md](../13-customer-success/CUSTOMER-ONBOARDING.md) flow written, using [CUSTOMER-ONBOARDING-TEMPLATE.md](../10-templates/CUSTOMER-ONBOARDING-TEMPLATE.md).
- [ ] [SALES-TO-CUSTOMER-HANDOFF.md](../07-sales/SALES-TO-CUSTOMER-HANDOFF.md) confirmed to transfer full context, not just a signed contract.

## Phase 6 — Metrics and Cadence

- [ ] Each active domain's metrics doc has at least one real baseline number, not a placeholder.
- [ ] [OPERATING-CADENCE.md](../01-business/OPERATING-CADENCE.md) states an actual review rhythm, and the first review is on a calendar.

## Before Calling This "Done"

This checklist is complete when every checked box corresponds to a real, filled-in document — not a copied template with placeholders still in it (see [FOLDER-STRUCTURE.md](FOLDER-STRUCTURE.md), "templates are copied out, never filled in place"). See [COMPANY-OPERATING-PACKAGE.md](COMPANY-OPERATING-PACKAGE.md) for the minimum artifact set this checklist should produce.
